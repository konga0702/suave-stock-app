# 在庫運用ガイド（帳簿優先 + 個体補正）

このガイドは、現在の運用（帳簿優先）で在庫ズレを減らしながら日々の出荷を止めないための手順です。

## 1. まず見る場所

1. `在庫` 画面で商品を開く  
2. `純在庫` タブを開く  
3. 以下の3つを確認する
   - `帳簿純在庫`（入庫 - 出庫）
   - `実在庫`（`inventory_items` の `IN_STOCK` 件数）
   - `不整合件数`（管理番号ごとの差分件数）

## 2. 表示の意味

- `帳簿純在庫`: 取引データ上の在庫数
- `実在庫`: 個体データ上の在庫数
- `不整合`: 帳簿と個体データが一致していない管理番号

`Supabase個体データ（全件可視化）` の見方:

- `総個体数`: その商品の個体レコード総数
- `IN_STOCK`: 在庫として残っている個体
- `SHIPPED`: 出庫済み個体

## 3. 不整合カードの判断ルール

### A. `実在庫 -N`（足りない）

- 意味: 帳簿にはあるが個体が足りない
- 現物がある: `実物ありで補正`
- 現物がない: `実物なしで補正`

### B. `実在庫 +N`（余っている）

- 意味: 帳簿より個体が多い
- 現物がある: `実物ありで補正`
- 現物がない: `実物なしで補正`

## 4. 実務での基本ルール（迷ったらこれ）

- 現物を正とする
- 管理番号が「ある / ない」で補正ボタンを選ぶ
- 補正後に同じカードが消えるか確認する

## 5. 日次チェック（1-2分）

- 今日出荷した商品の `純在庫` タブだけ確認
- `不整合件数` が増えていないか確認
- 増えていたらその場でカード補正

## 6. 週次 or 月次チェック

- `月次一括棚卸` を使う前に、現物確認が必要な商品だけ先に補正
- 一括処理後、主要商品の `帳簿純在庫` と `実在庫` が揃っているか確認

## 7. 今後ズレを減らす入力ルール

- 入庫時: 管理番号を必ず登録
- 出庫時: 管理番号を必ず確認して処理
- 管理番号の表記ゆれを避ける（ハイフン / 大文字小文字）

## 8. 重要な前提

現在は「業務を止めない」ため、在庫個体が足りなくても帳簿だけ更新できる設定です。  
そのため、ズレが出た場合は `純在庫` タブの不整合カードで後追い補正する運用が前提です。
